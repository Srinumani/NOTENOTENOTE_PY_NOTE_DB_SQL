Now question:

👉 How do we combine employee details with department location?

Answer: JOIN.


Inner Join
--------------

Returns only matching records from both tables.

SELECT e.name, e.salary, d.location
FROM employees e
INNER JOIN departments d
    ON e.department = d.dept_name;

Left join
--------------

All rows from LEFT table

Matching rows from RIGHT table

If no match → NULL

SELECT e.name, d.location
FROM employees e
LEFT JOIN departments d
    ON e.department = d.dept_name;

Used when:
You want all employees, even if department missing.

Right join
---------------

RIGHT JOIN

Opposite of LEFT.

SELECT e.name, d.location
FROM employees e
RIGHT JOIN departments d
    ON e.department = d.dept_name;

Usually avoided in real projects (LEFT is preferred).

FULL JOIN
__________________


All records from both tables.

(Only supported in some DBs like PostgreSQL, not MySQL directly)



Window function 
---------------------------

It performs calculation:

Across a group of rows                        

But WITHOUT collapsing rows like GROUP BY

👉 That is the key difference.

4, 5

RANK()

DENSE_RANK()

Gives:
4, 4
Next becomes 5 (no skipping)

Window Frame Clause (VERY IMPORTANT)

This controls how many rows the function sees.

Default Frame (when ORDER BY used)
RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW

This means:
Start → First row
End → Current row

Common Frame Types
🔹 Running Total
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
🔹 Moving 3-row Window
ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
🔹 Full Partition
ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING

CTE
------------------

CTE stands for common table expression its creating a temporary named result set

Instead of writing a long complex query inside another query,
you give it a name first.

It improves:

Readability

Debugging

Maintenance

Interview clarity

syntax 
----------
WITH cte_name AS (
    SELECT ...
)
SELECT * 
FROM cte_name;




Why Engineers Prefer CTE

Imagine 300-line query.

With subqueries → nested brackets → headache.

With CTE:

Step 1: Prepare data
Step 2: Calculate ranking
Step 3: Filter top results
Step 4: Final output

What is Apache Spark

Apache Spark is a framework that processes huge datasets across multiple machines.

Instead of one computer processing data:

1 computer → very slow

Spark splits work across many machines in parallel.

Spark Architecture (Simple View)

      ↓

      ↓
Tasks executed in parallel

Components:

Cluster	group of machines
